项目思路
让我AI列出来了针对现在代码做阶段性闭环的改进方向，特别提到让朋友用起来，有可以用云端，但是会触发微信风控，所以跑在本机，最终整理出需求如下

我要在 Mac 上开发一个 Go 程序，编译成 Windows 可执行文件发给朋友。朋友双击后，浏览器自动打开一个网页界面，他只需要粘贴链接点击下载，文章就会保存到本地，且全程不闪退、不报错、不乱码

针对需求，最终我们的思路如下

在 Mac 上利用 Go 的 embed 特性将 Web 界面打包进程序，编写自动唤起浏览器和 SSE 实时日志推送的逻辑，并在底层严格适配 Windows 的文件路径与命名规范，最终交叉编译出一个双击即用、零配置的单文件 EXE

针对这个思路拆分，我们接下来做

摒弃传统的命令行交互，利用 Go 的 embed 特性将 HTML 前端资源直接打包进二进制文件，并编写启动逻辑自动唤起用户默认浏览器，实现‘双击 EXE 即打开操作界面’的零门槛体验。”
